Форматы штрих-кодов

В Мастере штрих-кода можно выбрать любой из следующих 18 стандартных отраслевых форматов.

Codabar

Формат Codabar широко используется в библиотеках, на станциях переливания крови, а также при доставке посылок авиапочтой. Этот формат переменной длины поддерживает кодировку следующих 20 символов: 0123456789-$:/.+ABCD. Начальным и конечным знаками сообщения, закодированного в формате Codabar, должны быть буквы A, B, C или D.

Code 25

Code 25, известный также как «Code 2 из 5», – это дискретный формат переменной длины для кодирования цифровых знаков. В формате Code 25 каждый кодируемый знак представлен пятью полосами, две из которых широкие. Он используется в основном для маркировки товаров на складах, а также для идентификации фотоконвертов, авиабилетов, багажа и грузов.

Code 39

Формат Code 39, известный также как «Code 3 из 9», является наиболее распространенным форматом штрихкода, который используется в оптовой торговле для маркировки и отслеживания движения товаров. В формате Code 39 каждый кодируемый знак представлен девятью элементами (полосами или интервалами), три из которых широкие. Этот штрих-код широко применяется в промышленности, военном деле и медицине. Этот дискретный формат переменной длины поддерживает кодирование следующих 43 символов:

0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZ-.*$/+%

Звездочка (*) может использоваться только в качестве начального/конечного знака и не может стоять в основной части штрих-кода.

Можно также добавить контрольную цифру, которая помогает обеспечить безопасность штрих-кода. Code 39 поддерживает форматы контрольных цифр Modulo 43 и xxx-nnnnnnn-c, используемых таможенной службой США для контроля экспортно-импортных поставок.

Code 128

Code 128 – это высокоплотный формат переменной длины для кодирования буквенно-цифровых знаков. Область его применения – транспортировка и маркировка товаров. Данный код представляет собой 106 комбинаций полос и интервалов. Каждая комбинация имеет одно из трех значений, зависящее от того, какой из трех наборов знаков задействован. Первый набор знаков используется для кодирования всех знаков верхнего регистра и управляющих символов ASCII; второй – для кодирования всех знаков верхнего и нижнего регистров; третий – для кодирования числовых пар от 00 до 99. Используемый набор знаков определяется начальным знаком.

В формате Code 128 можно также создавать следующие четыре функциональных кода.

•
 
FNC1 используется в Европейском товарном коде (EAN).
•
 
FNC2 дает устройству считывания штрихкода команду связать сообщение символа штрихкода с сообщением текстового символа.
•
 
FNC3 дает устройству считывания штрихкода команду выполнить сброс.
•
 
FNC4 используется в закрытых системах.

Разновидностью формата Code 128 является EAN-128. Для этого символа применяется тот же набор знаков, что и в формате Code 128, однако функциональные коды FNC2 и FNC4 использоваться не могут, а код FNC1 может использоваться только как часть начального кода. С помощью дополнительного параметра для Code 128 в Мастере штрих-кода можно включить или отключить формат EAN-128. Code 128 полностью поддерживается текстовым форматом ASCII.

EAN-8

Европейский товарный код (EAN) – это европейский аналог универсального кода продуктов (UPC). Теперь этот код называется международным товарным кодом, однако аббревиатура EAN по-прежнему используется. Коды EAN используются для маркировки товаров, реализуемых в розничной торговле стран Европы.

Код EAN-8 состоит из восьми цифр: первые две цифры означают код страны, следующие пять цифр – основные данные, а последняя является контрольной цифрой. В Мастере штрих-кода необходимо ввести семь цифр, а восьмая цифра (контрольная) генерируется автоматически.

К основному штрих-коду можно добавить дополнительную двух- или пятизначную цифру. Эта цифра предназначена для публикаций и периодических изданий и отображается в качестве дополнительного штрих-кода справа от основного.

EAN-13

EAN-13 – это европейский аналог универсального кода продуктов (UPC (A)). Разница между EAN-13 и UPC (A) заключается в том, что в коде EAN-13 тринадцатая цифра кодируется как шестая слева цифра символа UPC (A). Тринадцатая цифра вместе с двенадцатой представляют код страны.

К основному штрих-коду можно добавить дополнительную двух- или пятизначную цифру. Эта цифра предназначена для публикаций и периодических изданий и отображается в качестве дополнительного штрих-кода справа от основного.

FIM

Наружные идентифицирующие метки (FIM) используются почтовой службой США для автоматической обработки писем. Метки FIM используются для автоматического распознавания лицевой стороны и отслеживания почтовых отправлений, на которых отсутствует марка или штамп, например, на деловых ответных письмах, почтовых квитанциях о наложении штрафов и т.д.

В настоящее время используются следующие четыре типа меток FIM.

•
 
FIM-A используется для бесплатных ответных почтовых отправлений, снабженных предварительно напечатанным штрихкодом POSTNET.
•
 
FIM-B используется для почтовых отправлений, таких как деловые ответные письма, почтовые квитанции о наложении штрафов, официальная (франкированная) корреспонденция, не снабженных предварительно напечатанным штрихкодом POSTNET.
•
 
FIM-C используется для почтовых отправлений, таких как деловые ответные письма, почтовые квитанции о наложении штрафов, официальная корреспонденция, снабженных предварительно напечатанным штрихкодом POSTNET.
•
 
FIM-D указывает на необходимость оплаты почтовых услуг.

Метки FIM печатаются в правом верхнем углу параллельно верхнему краю конверта на расстоянии пяти сантиметров от правого края.

ISBN

Коды ISBN (международные стандартные номера книг) печатаются на книгах. Формат ISBN не является отдельным видом штрих-кода. Коды ISBN имеют особую структуру и представляют собой штрих-коды, кодированные в формате EAN-13. Штрих-код включает код страны 978, состоящий из трех цифр, за которым следует десятизначный номер ISBN. Десятая цифра (контрольная) отбрасывается. Цифры вводятся в формате x-xxxx-xxxx. Коды ISBN должны быть числовыми.

Если требуется создать 13-значный штрих-код ISBN, см. раздел Создание 13-значного штрих-кода ISBN.

ISSN

Коды ISSN (международные стандартные номера периодических изданий) печатаются на журналах, газетах и других периодических изданиях. Данный формат не является отдельным видом штрих-кода. ISSN – это штрих-код EAN-13, включающий код страны 977 и дополнительный двузначный код. Дополнительный двузначный код означает номер выпуска, например: январь = 01, февраль = 02. Цифры вводятся в формате xxxx-xxxx. Коды ISSN должны быть числовыми.

ITF

Код ITF (Interleaved 2 из 5) был создан на основе кода Code 25. В этом формате используется те же способы кодировки, только полосы и интервалы содержат данные в формате ITF: нечетные (по порядку) знаки представлены полосами, а четные – интервалами. ITF – это высокоплотный формат переменной длины для кодирования только цифровых знаков. Такой штрих-код является одним из наиболее распространенных форматов, используемых при транспортировке и складском хранении товаров. В Мастере штрих-кода можно ввести до 40 знаков. Коды ITF должны быть числовыми.

ITF-14

Формат ITF-14 похож на ITF, только для него в Мастере штрих-кода следует вводить строго 13 знаков. Можно ввести дополнительные пять знаков в качестве дополнительного штрих-кода. Коды ITF-14 должны быть числовыми.

JAN-8

JAN-8 – это японский эквивалент кода EAN-8. Коды Jan-8 должны быть числовыми.

JAN-13

JAN-13 – это японский эквивалент кода EAN-13. Коды Jan-13 должны быть числовыми.

MSI Plessey

Штрих-код MSI Plessey используется, главным образом, в библиотеках и для маркировки складских стеллажей. MSI Plessey – это формат переменной длины, позволяющий кодировать следующие 10 знаков: 0123456789. Каждый знак состоит из восьми элементов: четырех полос и четырех интервалов.

Pharmacode

Pharmacode используется для интерактивного контроля безопасности процесса упаковки фармацевтической продукции. Данные содержатся только в полосах, интервалы информации не несут. Формат Pharmacode является очень практичным благодаря высоким допускам по точности печати, а также возможности цветной печати штрих-кодов. Коды в формате Pharmacode должны быть числовыми.

POSTNET

Штрих-коды POSTNET (способ цифрового кодирования почтовых отправлений) используются почтовой службой США для кодирования почтовых индексов. Процесс обработки корреспонденции почтовой службой полностью автоматизирован, и штрих-коды POSTNET считываются автоматическим оборудованием.

Формат POSTNET отличается от других форматов тем, что здесь изменяется высота, а не ширина полос. Каждая цифра представлена комбинацией из пяти полос. В качестве начальной и конечной полос используется одна высокая полоса. Формат POSTNET может использоваться для пяти-, девяти- и 11-значных штрих-кодов адреса доставки. Эти коды часто используются совместно с кодами FIM, расположенными в правом верхнем углу почтового отправления, например на деловых ответных письмах. Коды Postnet должны быть числовыми.

UPC(A)

Коды UPC (универсальные коды продуктов) применяются в розничной торговле США и Канады. UPC(A) – это 12-значный формат. Код состоит из 11 цифр, содержащих основные данные, и одной контрольной цифры. Первая цифра обычно обозначает тип идентифицируемого продукта. Следующие пять цифр означают код производителя, а последние пять содержат информацию о конкретном продукте. Коды UPC(A) должны быть числовыми.

UPC(E)

Подобно коду UPC(A), код UPC(E) применяется в розничной торговле. Однако размер этого штрих-кода меньше, поэтому он удобен для маркировки продуктов небольшого размера. Данный формат также называют «отбрасывающим незначащие нули», поскольку в UPC(E) 12-значный код UPC(A) сжимается до шестизначного. В UPC(E) отбрасывается знак системы нумерации, замыкающие знаки кода производителя и начальные нули в части штрих-кода, содержащей информацию о продукте.

При необходимости к основному штрих-коду UPC(A) или UPC(E) можно добавить двух- или пятизначную цифру. Эта цифра предназначена для публикаций и периодических изданий и отображается в качестве дополнительного штрих-кода справа от основного. Коды UPC(E) должны быть числовыми.
